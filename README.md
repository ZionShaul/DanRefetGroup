# משקי דן — ניהול רכש רפתות

אפליקציית ווב (מובייל-first, עברית/RTL) שמציגה לכל רפת את **יתרות הרכש שלה** מתוך קובץ הספקים
המאוחד. כל רפת רואה רק את הנתונים שלה: כרטיס יתרה לכל חומר (שם פריט), ולחיצה על חומר פותחת את
הפירוט המלא (לפי חודש אספקה וספק, כולל שורות ההזמנה).

- **Frontend/Backend:** Next.js 16 (App Router) + React 19 + Tailwind v4
- **DB + Auth:** Supabase (Postgres, Auth עם קוד חד-פעמי לאימייל, RLS)
- **פריסה:** Vercel
- **PWA:** הוספה למסך הבית, פתיחה במסך מלא, מיתוג ירוק של משקי דן

## תהליך הנתונים (שלב 1 — טעינה ידנית)

1. במחשב המקומי מכינים את **קובץ הספקים המאוחד** (`merged_suppliers_*.xlsx`) — איחוד אמבר/מילובר/IDM
   (פרויקט `amber-automation`).
2. מנהל מעלה את הקובץ במסך **ניהול ← טעינת קובץ**. המערכת מפרסרת, מפצלת לפי עמודת **ארגון** (רפת),
   ויוצרת אוטומטית רפתות חדשות שזוהו.
3. המנהל בוחן את התצוגה המקדימה ולוחץ **פרסום**. הפרסום מחליף את הנתונים הקודמים (מצב "החלפה בכל טעינה").

## תפקידים

- **משתמש רגיל (רפת):** רואה רק את היתרות והשורות של הרפת שאליה הוא משויך, מהטעינה הפעילה בלבד (נאכף ב-RLS).
- **מנהל:** רואה הכל, בוחר רפת לצפייה, מנהל משתמשים ורפתות, וטוען/מפרסם קבצים.

ריבוי משתמשים לאותה רפת נתמך (כל משתמש משויך ל-`organization_id`).

## מבנה

| נתיב | תיאור |
|------|-------|
| `app/(app)/page.tsx` | מסך הבית — כרטיסי יתרה לכל חומר (למנהל: בורר רפתות) |
| `app/(app)/product/[key]/page.tsx` | פירוט חומר — סיכום לפי חודש/ספק + שורות הזמנה |
| `app/admin/upload` | טעינת קובץ מאוחד → תצוגה מקדימה → פרסום |
| `app/admin/users` | ניהול משתמשים ושיוך לרפתות |
| `lib/excel/parse.ts` | פרסור עמודות הקובץ המאוחד |
| `lib/orders.ts` | שאילתות יתרה/שורות לרפת |
| `supabase/migrations/0001_init.sql` | סכמה, RLS, RPC |

ראו **SETUP.md** להקמת Supabase, משתני סביבה, יצירת מנהל ראשון ופריסה ל-Vercel.

## פיתוח מקומי

```bash
npm install
cp .env.local.example .env.local   # מלא/י פרטי Supabase
npm run dev
```
